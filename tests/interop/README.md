# Cross-SDK signature interoperability tests

These tests prove that a signature generated by **any Quidnug SDK**
verifies against **every other Quidnug SDK** — and against the Go
reference node — byte-for-byte.

Without this, a subtle divergence in canonical-bytes production (e.g.
a different JSON escaping rule, a stray trailing newline, a missed
nested-key sort) would silently cause every signature made in one
language to be rejected by nodes running any other language's
verifier. That's the kind of bug that's catastrophic to discover in
production.

## How it works

```
 ┌───────────────────┐     ┌──────────────────────────────┐
 │ Python signs      │────▶│ test vectors (JSON + sig)    │
 │ Go signs          │     └──────────────────────────────┘
 │ Rust signs        │                  │
 │ Java signs        │                  ▼
 │ .NET signs        │     ┌──────────────────────────────┐
 │ JS signs          │────▶│ Each SDK verifies every other│
 │ Swift signs       │     └──────────────────────────────┘
 └───────────────────┘
```

Each language produces a **canonical signable bytes + signature +
public key** bundle over 4 standard test transactions. A shared test
harness in every SDK verifies all the bundles it can read.

## Test vectors

The file `vectors.json` is the shared wire format:

```json
{
  "sdk": "python|go|rust|java|dotnet|js|swift",
  "version": "2.0.0",
  "cases": [
    {
      "name": "trust-basic",
      "tx": { "type": "TRUST", ... },
      "excludeFields": ["signature", "txId"],
      "canonicalBytesHex": "7b227479...",
      "publicKeyHex": "04...",
      "signatureHex": "3045..."
    }
  ]
}
```

Every SDK produces its own `vectors-<sdk>.json`. The harness loads
all present files and cross-verifies every `(producer, verifier)` pair.

## Running

```bash
# Produce vectors in every language
cd tests/interop
make produce     # runs Python, Go, Rust, Java, .NET, JS, Swift
make verify      # each language verifies every other
```

Or piecewise:

```bash
python produce.py > vectors-python.json
go run ./produce > vectors-go.json
cargo run --manifest-path produce-rs/Cargo.toml > vectors-rust.json
# ...

# Python verifier checks every vector file
python verify.py vectors-*.json
```

## Failure modes caught

- Canonical bytes differ between languages (key-order, escaping,
  number-formatting, Unicode normalization)
- Signature DER encoding differs (strict vs. lax, leading-zero
  padding)
- Public-key encoding differs (SEC1 vs. compressed)
- Quid ID derivation differs (hash algorithm, byte slice)

If any of these differ, the corresponding pair fails.

## CI

The CI matrix runs `make produce` on every push and posts the
resulting `vectors-*.json` as workflow artifacts. A nightly job
downloads them all and runs `make verify` across every pair. A red
result blocks release.

## Current status

Initial scaffold ships Python and Go producers + verifiers. Adding
a new language requires only:

1. A `produce.<lang>` script that emits `vectors-<lang>.json`.
2. A `verify.<lang>` script that accepts `vectors-*.json` and
   returns exit 0 iff every vector's signature verifies against
   the included public key.

See `produce.py` / `verify.py` as the reference pattern.
